
# アプリ名
・ManyMenu

# 制作背景(意図)
  ・コロナ禍で、飲食店は今年4月に９割以上の売上減少など大打撃を受けていました。
  この未曾有の危機的な状況を生き残るにはどうすれば良いのか。
  現状の問題解決に必要な物は何かを考え、今回のメニュー限定表示のアプリを制作しました。

# 概要
  ・各店舗でメニューを１２品限定で掲載するアプリです。（メニュー掲載サイト）<br>
  ・上記に加え、衛生面や空き時間も掲載することのできるアプリです。衛生面、空席時間の可視化サイト）


# 実装機能
・ユーザー管理機能
・検索機能
・新規投稿、編集機能
・計算機能

# ペルソナ

### ペルソナ（一人目）
##### ４０代男性飲食店店長
##### 趣味　野球観戦
##### 店は７０坪９０席を誇る大手飲食店の店長を担当しているAさん

・Aさんは店長として、お店の収益を向上させる役目を負っているが、昨今の新型ウイルスによって１日最高８０万を誇っていた売り上げを８割近く落としている。

#### 原因
・店内という密閉空間を顧客が避け始めたことに加えて、各行政府からの時短営業等を勧められたことにある。お店はアルバイトも解雇しても損益分岐点を大きく割り込むほどの窮地に立たされている。そのため本部からは売り上げ回復の案を考え各店舗予算内で実行せよとの一部裁量権は頂いたが、１０万円以内と資金面はあまりに乏しく広告やチラシ等打てる余裕はなかった。



#### 解決したい課題等
・資金面を押さえた上で落ち込んだ顧客を呼び戻したい。<br>
・また欲を言えばリピーターを増やし、既存客による収益から経営基盤を安定させたい。<br>
↓ <br>
・顧客の不安要素を取り除けば客足は回復の目処がたちのではないか。<br>
・満足度の上がる工夫を取り入れれば良いのではないか。<br>
↓
・顧客の不安要素のある、何かを取り除けるシステムが欲しい.<br>
・顧客の満足度の上がる何かが欲しい。

#### これがあれば良いと考えた案
・衛生面の可視化、空き時間の可視化、共通接触物の排除、マイ箸等の持参物の推進等。<br>
・行動経済学、選択のパラドックスの考え方を応用し、満足度向上をする工夫を追加すること。

その後緊急事態の解除とのことで、久々に息子たちと外食に出ようと考えたが、漠然とした不安は残っており飲食店に行けずにいた。<br>
目に見えない恐怖からテイクアウトや配達を選んではいたもののテイクアウトは費用も嵩み、子供たちも飽きてきたという理由から外食に切り替えたいと考えていた。

↓<br>

↓<br>
衛生面の可視化<br>


#### これがあれば良いと考えた案
・衛生面の数字での可視化、客足の混雑具合の可視化。




# アプリ制作で工夫したポイント
  ・事業者のみがログインを行います。本物の事業者かという判断は営業許可証を写真で送ってもらい行います。<br>
  ・アプリ運営側の収益を広告に依存せず、収益面のリスクを分散するために、QRコードは低価格ながら有料化しました。<br>


# このサイトのお客様の価値とは
1,メニューがまとめられていてシンプルで見やすい。<br>
2,各お店のカウンターやテーブルに設置してもらうことで、そこにあるQRコードから直接サイトへ飛べる。<br>
3,メニュー表示方法が統一されていて、他の店舗を閲覧していても既視感による扱いやすさを実現。<br>
4,目に可視化されづらい、衛生面に対する店舗側の努力が可視化されている。<br>
5,混雑具合が表示されており、人混みを回避して向かうことが可能になる。<br>
6,全員が触るであろうメニュー表を触らずに済み、感染リスクを下げることができる。また不安要素の排除に繋がる。<br>


# このサイトの事業者側の価値とは
7,メニュー表を印刷する量が減り、衛生面で毎度行う消毒等の機会も減り、コストカットにつながる。<br>
8,メニュー表示を工夫することで客単価の増加を見込める。<br>
9,使用料金は無料なため、使用するハードルが低い。<br>
10,感染症を不安に思う方の層の一部回復。<br>
11,メニューの記載も面倒であれば代行していただける。<br>


## 制作前には想定しなかった付加価値
12,利用者が増加すればメニュー表を回すという機会が減り、それぞれがメニューを見出すことから、今まで足し算であったメニューを選ぶ時間は、最大値のみとなりお客様の回転率をあげることにつながる。

# このアプリを制作後に感じた課題。
  ・現在のアプリでは、まずお客様の体感型ではないため、依存性が低いことによるアプリ継続利用者が見込めるかが不安視されます。<br>
  ・またWEBデザイン技術がまだ乏しいため、惹かれるような画面にできていないこと。<br>
  ・技術力はまだ途上にある分、動作するにおいて細かい事項に対応できない状態にあります。（現状ではメニューを１２品未満の表示にはできない。）<br>
  ・店主がQRコードまでダウンロードすることで成り立つものであり、そこまでの動作を踏んでくれるかが不安視されます。<br>
